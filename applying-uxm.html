Developing applications

To develop applications that comply fully with the nidirect UXM, we recommend that developers use the HTML/CSS and JS templates and examples provided in the UXM starter kit.

If developers wish to use their own html/css framework other than that provided in the UXM they must comply with the following principles:

  • The design of the UXM, in terms of the layout, structure (HTML), styling (CSS) and behaviour (JS), must be matched as closely as possible
  • Developers must use the UXM as a reference point. Use the developer tools in your favourite browser to inspect the HTML/CSS/JS of the UXM and also refer to guidance for UXM controls and design patterns. It is unlikely that direct support will be provided
  • Any deviation from the UXM visual design as presented here and in the UXM starter kit will be treated as a defect unless the deviation has been planned and agreed with the nidirect Transactional Services Manager
  • This UXM should also be applied to Commercial Off The Shelf (COTS) products, where possible. Advice should be sought from the nidirect Transactional Services Manager prior to commencing design changes

Twitter Bootstrap

This new version of the UXM is closely built upon Twitter Bootstrap v3.3.7. While not all components available in Twitter Bootstrap are presented within the UXM, the components available for nidirect transactional services can be found on the various 'Controls' pages.

The list of components and alignment with Twitter Bootstrap is expected to grow in future versions of the UXM, but specific requests can be made to the nidirect Transactional Services Manager when considering using a control from Bootstrap that is not presented within the UXM.

Browser support

Transactional services built for nidirect must be tested and be fully compliant on the following browsers:

Desktop

  • Internet Explorer 11+ - Windows
  • Google Chrome (latest version) - Windows and Mac OS X
  • Mozilla Firefox (latest version) - Windows and Mac OS X
  • Apple Safari (latest version) - Windows and Mac OS X

Small screen devices

  • Apple Safari (latest version) - iOS 6 and iOS 7
  • Google Chrome (latest version) - Android 4.x
  • Android browser - Android 4.x

This list is based on statistics from www.nidirect.gov.uk and represents over 95% of all desktop browsers. The remaining browsers are individually insignificant. Where 'latest version' is listed, it means the latest stable version plus one version back, as these browsers regularly self-update.
