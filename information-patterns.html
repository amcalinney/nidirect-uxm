Summary information

  • Uses description lists to display snippets of information about the user

    • <h4 id="list-summaryInformation">About the applicant</h4>

Application summary

  • An application summary is used to display all the information that a user has entered
  • The title of the summary should not given the impression that the information has already been submitted eg use "Review your answers"
  • The summary must not display sections which have been skipped within the application - or questions that have not been answered
  • The information displayed must displayed in the order that it was provided and must not be concatenated
  • Depending on the length of the application and the amount of information being presented in the summary, developers should decide on whether to present the summary information using an accordian control

Error pages

  • every error, regardless of who is to blame, results in a bad user experience
  • effectively designed error pages can help to nullify that experience
  • error messages should sound like they are written by humans - for humans
  • where possible, an error page should give the user options on what to do next

Not found (404) error message

The 404 or 'not found' error message is an HTTP standard response code indicating that the client was able to communicate with a given server, but the server could not find what was requested. The web site hosting server will typically generate a "404 Not Found" web page when a user attempts to follow a broken or dead link

Internal server (500) error message

The 500 or 'internal server' error message is an error generated by the Web server. It basically means something has gone wrong, but the server cannot be more specific about the error condition in its response to the client. In addition to the 500 error notified back to the client, the Web server should generate some kind of internal error log which gives more details of what went wrong.

Session error message

The session error message will appear after a defined period of client inactivity, or in cases when the user attempts to click the back button after a transaction is complete. Guidance provided by OWASP on session expiration(external link opens in a new window / tab) recommends that the period of client inactivity should be dependent on how critical the web application and its data are, and can range from 2 to 5 minutes for high-value applications to 15 to 30 minutes for low risk applications.

General error message

The general error message is designed to cover 'catch-all' errors not covered by either the 404 or 500 error messages.
